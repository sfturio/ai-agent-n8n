AI Agent

Technical documentation for the Node.js + n8n assistant

PT-BR
Repository Live App

Overview

AI Agent is a lightweight developer assistant focused on log analysis, code-oriented questions, and operational context. The backend is modular and integrates with n8n via webhook. Interaction history is stored in Supabase when configured.

Architecture

UI (public/) -> POST /api/agent -> controller -> service -> n8n webhook
                          \-> saveInteractionLog -> Supabase

API

POST /api/agent

Send a user message to the assistant.

{
  "message": "Explain this timeout"
}
  • 200 text/plain: assistant reply (includes fallback when n8n is unavailable)
  • 400 application/json: invalid input
  • 500 application/json: unexpected internal error

GET /health

{ "ok": true }

Environment Variables

PORT=3000
N8N_WEBHOOK_URL=https://your-n8n-instance/webhook/agent
SUPABASE_URL=https://.supabase.co
SUPABASE_SERVICE_ROLE_KEY=
SUPABASE_TABLE=agent_messages

Local Setup

  1. Install dependencies: npm install
  2. Configure .env from .env.example
  3. Run app: npm run dev
  4. Open http://localhost:3000

Supabase Table

create table if not exists public.agent_messages (
  id bigserial primary key,
  created_at timestamptz not null default now(),
  message text,
  response_body jsonb,
  ok boolean not null default false,
  error_message text
);

Troubleshooting

  • 500 on /api/agent: check N8N_WEBHOOK_URL and n8n webhook status.
  • No saved history: check Supabase credentials (SUPABASE_URL and service key).
  • Render boot issues: confirm npm start and Node version >= 18.
